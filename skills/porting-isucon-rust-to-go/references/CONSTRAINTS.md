# Hard Constraints (Disqualification Risks)

These four items are **immutable** for the duration of the contest. Touching any of them causes disqualification regardless of score. Every subagent prompt generated by this skill must embed this list verbatim.

## 1. API URI and Request/Response Structure

**Rule:** The Go port must expose the exact same set of (METHOD, path) pairs as the Rust reference, and every endpoint must accept the same request shape and return a functionally and structurally equivalent response.

**Concretely:**
- HTTP method must match.
- Path including path parameter names must match (`/api/livestream/:livestream_id` not `:id`).
- Query parameter names, types, and required-vs-optional must match.
- Request body field names, types, and `Content-Type` must match.
- Response body field names, types, ordering (where the benchmarker checks ordering), and `Content-Type` must match.
- HTTP status codes for each branch (200, 201, 204, 400, 401, 403, 404, 409, 500) must match.
- Headers the Rust app sets (cookies, `Cache-Control`, `Location` on 3xx) must be set identically.

**Common porting traps:**
- Rust `serde(rename_all = "camelCase")` — every field needs a matching `json:"camelCase"` tag in Go.
- Rust `Option<T>` serialising as `null` vs being omitted — `serde(skip_serializing_if = "Option::is_none")` maps to `omitempty` in Go.
- Rust `chrono::DateTime<Utc>` serialised as RFC3339 — Go `time.Time` uses RFC3339Nano by default; explicit format matching may be needed.
- Rust integer types: `i64` → Go `int64`, `i32` → Go `int32`. Wrong width = wrong JSON in some benchmarkers' eyes.
- Rust `BigDecimal` / `Decimal` — must port to a string-preserving type, not float, or precision drifts.

## 2. Frontend Static Files

**Rule:** HTML, CSS, JavaScript, images, fonts, and any other distributed static asset must be served byte-for-byte as distributed.

**Concretely:**
- Do not regenerate, minify, recompress, or "fix" any static file.
- Serve from the same URL paths the Rust app does (usually `/` and/or `/assets/...`).
- `Content-Type` must match (browsers and the benchmarker both check).
- If the Rust app serves index.html as the SPA fallback, the Go app must do the same fallback for the same path patterns.

**Common porting traps:**
- Echo's `Static()` and `File()` middleware sometimes alter trailing-slash behaviour vs the Rust `tower-http::services::ServeDir`. Test the SPA fallback path explicitly.
- `Cache-Control` and `ETag` behaviour: prefer to mirror the Rust app exactly. Echo's defaults may add headers the Rust app does not.

## 3. isuwari (Reboot/Retest Control Daemon) and Related Files

**Rule:** Do not modify, move, replace, disable, or interfere with `isuwari` or any file it owns.

**Concretely:**
- `/opt/isuwari/` and any binary/config under it.
- Any systemd unit named `isuwari*` (`isuwari.service`, `isuwari-*.timer`, etc.).
- Any cron, sudoers, or helper script `isuwari` invokes.
- Anything `isuwari` writes to during retest.

**Verify untouched:**
```bash
sudo systemctl status isuwari
ls -la /opt/isuwari/ /etc/systemd/system/isuwari*
```

If a port step would create files under those paths, **stop and ask** — do not proceed.

## 4. isuadmin User Account

**Rule:** Do not modify the `isuadmin` user's account, group memberships, sudoers entry, SSH authorized_keys, password, or login configuration.

**Concretely:**
- Do not change `/etc/passwd`, `/etc/shadow`, `/etc/group` entries for `isuadmin`.
- Do not edit `/home/isuadmin/.ssh/authorized_keys` or any dotfile in that home.
- Do not add or remove `isuadmin` from any group.
- Do not change any sudoers entry referencing `isuadmin`.

**Verify untouched:**
```bash
getent passwd isuadmin
sudo grep isuadmin /etc/sudoers /etc/sudoers.d/* 2>/dev/null
sudo ls -la /home/isuadmin/.ssh/
```

## What Is Allowed

For the avoidance of doubt, these are explicitly fair game:
- The application code under `webapp/` (rust/, go/, etc.).
- nginx config, MySQL config, sysctl, app systemd units (other than `isuwari*`).
- Everything in the `isucon` user's home.
- Adding new systemd units for the Go port (e.g. `isuride-go.service`).

The Phase 2 subagent generator copies the constraint summary (sections 1–4) into every `port-*` subagent's system prompt. Do not paraphrase — copy verbatim. Subagents running in fresh contexts will not have access to this file unless it is inlined into their prompt.
